# СИСТЕМА УЧЁТА СОТРУДНИКОВ ОРГАНИЗАЦИИ

## Описание проекта

Консольное Java-приложение для управления данными о сотрудниках организации с использованием:
- Потоков выполнения (Thread, Runnable)
- Коллекций (ArrayList)
- Файлового ввода-вывода
- Обработки исключений

## Структура проекта

```
├── Employee.java          - Класс модели сотрудника
├── EmployeeManager.java   - Управление коллекцией сотрудников
├── FileSaveTask.java      - Поток для сохранения данных
├── FileLoadTask.java      - Поток для загрузки данных
└── Main.java              - Главный класс с меню
```

## Как запустить программу

### 1. Компиляция

```bash
javac *.java
```

### 2. Запуск

```bash
java Main
```

## Функциональность

1. **Добавление сотрудника** - добавление нового сотрудника в систему
2. **Просмотр всех сотрудников** - отображение списка всех сотрудников
3. **Поиск сотрудника** - поиск по ФИО (поддерживает частичное совпадение)
4. **Сохранение в файл** - сохранение данных в `employees.txt` (в отдельном потоке)
5. **Загрузка из файла** - загрузка данных из `employees.txt` (в отдельном потоке)

## Формат файла

Данные хранятся в текстовом файле `employees.txt`, где каждая строка содержит информацию об одном сотруднике:

```
Иванов Иван Иванович;35;Программист;80000.0
Петрова Мария Сергеевна;28;Менеджер;65000.0
```

Формат: `ФИО;Возраст;Должность;Зарплата`

## Особенности реализации

### Использование потоков
- Сохранение и загрузка данных выполняются в отдельных потоках
- Используется интерфейс `Runnable`
- Главный поток ожидает завершения операций через `join()`

### Работа с файлами
- Используется `try-with-resources` для автоматического закрытия ресурсов
- Обработка `IOException` и `NumberFormatException`
- Автоматическое создание файла при сохранении

### Коллекции
- Используется `ArrayList<Employee>` для хранения данных
- Перебор элементов через `for-each`

### Обработка исключений
- Все ошибки ввода-вывода обрабатываются через `try-catch`
- Валидация пользовательского ввода
- Информативные сообщения об ошибках
